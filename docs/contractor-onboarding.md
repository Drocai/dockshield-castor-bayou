# Contractor Onboarding

## Before Access

Confirm:

- scope of assignment
- budget or rate
- delivery date
- ownership terms
- asset license terms
- whether source files are included
- whether commercial use is allowed

## Access Level

Give the least access needed.

Preferred sequence:

1. share docs only
2. share asset brief
3. share private repo read access
4. grant write access only when needed

## Required Reading

Every contractor should read:

- `README.md`
- `AGENTS.md`
- `docs/00-start-here.md`
- `docs/core-gameplay-bible.md`
- `docs/castor-bayou-vertical-slice.md`
- the relevant brief in `docs/outsourcing-briefs.md`

## Delivery Format

Ask for:

- source files
- exported game-ready files
- license note
- short change summary
- known limitations

## Do Not Accept

- unlicensed assets
- unclear AI-generated assets
- ripped game assets
- marketplace assets from accounts not controlled by the project owner
- files with embedded secrets or unrelated content
